Durante las últimas iteraciones en `lotusia.org`, completamos una amplia actualización del sitio estático enfocada en el pulido del producto, la calidad del SEO, la corrección multilingüe y el enrutamiento seguro en producción.

Esta publicación resume qué cambió y por qué importa.

## Mejoras de UI y Navegación

Corregimos varios problemas de usabilidad en el sitio de marketing:

- Los menús desplegables del encabezado fueron reconstruidos para una interacción estable (comportamiento de hover/clic/foco), de modo que los menús ya no desaparecen al mover el cursor.
- El selector de idioma recibió etiquetas con banderas y renderizado sin saltos de línea para evitar el quiebre de texto.
- Los botones y chips de enlace se normalizaron con el estilo de la marca principal para una jerarquía visual consistente.
- Los iconos faltantes fueron auditados y restaurados en todas las páginas.
- El diseño del índice del blog fue corregido para que el primer artículo destacado se renderice como se pretende en pantallas grandes.

Estos cambios hacen que el sitio sea más fácil de navegar y reducen las inconsistencias visuales.

## Mejoras de SEO y Calidad de Contenido

También lanzamos mejoras prácticas de SEO que mejoran la calidad del rastreo y la estabilidad del renderizado:

- Se añadió manejo significativo de texto `alt` para imágenes editoriales y de contenido.
- Se añadieron `width`/`height` de imagen donde fue posible para reducir el [cambio de diseño (CLS)](https://web.dev/articles/cls).
- Se mejoró la generación de [`hreflang`](https://developers.google.com/search/docs/specialty/international/localized-versions) para evitar enlaces alternativos duplicados que apuntan a la misma URL en páginas no traducidas.
- Se añadió soporte de `lastmod` en el [sitemap](https://www.sitemaps.org/protocol.html) basado en fechas de contenido y tiempos de modificación de archivos.
- Se mantuvo la generación de URL canónica consistentemente vinculada a la configuración del dominio de producción.

En resumen: metadatos más limpios, mejores señales de indexación y mejor experiencia para usuarios y rastreadores.

## Internacionalización y Nueva Página de Servicios Beta

La nueva página `/beta-services` ahora forma parte del conjunto de rutas localizadas y se genera para todos los idiomas soportados.

Incluye enlaces directos a:

- `https://app.lotusia.org`
- `https://app.lotusia.org/social/activity`

La página ahora cuenta con `title`, `description` y texto hero localizado en todos los archivos i18n existentes, asegurando metadatos localizados adecuados y consistencia a nivel de página.

## Enrutamiento Edge Más Seguro Durante la Migración

Mover el dominio apex al alojamiento estático requirió un enrutamiento cuidadoso para que los servicios dinámicos existentes sigan funcionando.

Actualizamos las reglas de redirección/proxy en el edge para preservar la continuidad del servicio para rutas como:

- `/social/*`
- `/api/*`
- `/explorer/*`

Este enfoque híbrido mantiene la superficie de marketing estática y rápida mientras preserva el comportamiento de aplicaciones dinámicas a través de hosts de servicio dedicados.

## Despliegue y Operaciones

Para mejorar la confiabilidad y la repetibilidad:

- Se añadió un flujo de trabajo de GitHub Actions para validación de compilación y despliegue.
- Se añadió documentación de despliegue y listas de verificación para operadores.
- Se añadió un script de despliegue para el flujo de despliegue de Cloudflare específico por cuenta.

En conjunto, estos cambios hacen que los lanzamientos sean más seguros y fáciles de operar.
